## v0.2 — 品質・安定性 ✅ 完了

**目標**: Amazon の DOM 変更に強くする。日常使いできる完成度にする。

### DOM 耐性の強化

Amazon は DOM 構造を変更することがある。新パターンを発見したら即フィクスチャ化して回帰検証に追加する運用を継続する。

- [ ] fixtures を定期的に最新 DOM で更新する（目安: 月 1 回）
- [ ] detectSale が誤検知・見逃しを起こしたら `shared/detect.js` を修正 → `node verify-detect.mjs` で全件 PASS を確認
- [ ] 追加すべきパターン候補（DOM サンプルが必要）:
  - `a-badge` 系の新バッジ（現在未対応）
  - プライム会員限定価格の表示形式が変わった場合
- ~~クーポン~~ — **ウィッシュリスト DOM に出ない（Amazon 仕様）**。検出するには各商品ページへの個別 fetch が必要で、`host_permissions` 追加・レート制限リスクを伴うため保留

参照: [fixtures/HOW_TO_CAPTURE.md](fixtures/HOW_TO_CAPTURE.md)

### 既知の制約の解消

- [x] **アイコン差し替え** — サークル型デザインに更新（v0.2.0）

## v0.4 — 横断セール確認 ✅ 完了

**目標**: 単一リストではなく、ユーザーが持つ全ウィッシュリストを横断してセール商品を集約する。

- [x] **横断スキャン** — popup の「全リスト横断スキャン」から、サイドバーの全リストを
  バックグラウンドタブで順に開き、既存の全件読み込みロジックを再利用してセール判定（v0.4.0）
- [x] **専用結果ページ** — `results/results.html` にリスト別セール商品を集約表示（v0.4.0）
- [x] **エラー耐性** — 1リスト失敗時もそのリストを「エラー」表示にして残りを続行（v0.4.0）

設計メモ:
- バックグラウンドタブはタイマーがスロットリングされるため、ヘッドレススキャン時のみ
  `loadAllItems` の新規アイテム待ち時間を 600ms → 4000ms に延長
- 結果ページを途中で閉じたら `chrome.tabs.onRemoved` でスキャンを中断（デッドロック防止）
- 新規パーミッションなし（`tabs` も不要、tabId 追跡のみで完結）

## Chrome Web Store 公開に向けた準備

公開するには以下が必要。

| 作業 | メモ |
|------|------|
| アイコン正式化 | 16 / 48 / 128px、ストアの画像ガイドラインに準拠 |
| プロモーション画像 | 1280×800px のスクリーンショット（最低 1 枚） |
| プライバシーポリシー | 「収集するデータなし」でシンプルに書く。公開 URL が必要 |
| `host_permissions` 見直し | 現状は content script の `matches` のみ。fetch が必要になったら追加 |
| ストア説明文 | 日本語・英語の両方があると望ましい |

> 参考: [developer.chrome.com/docs/webstore/publish](https://developer.chrome.com/docs/webstore/publish/)
